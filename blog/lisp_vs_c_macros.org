* Lisp is homoiconic and that's a big thing
  As [[https://gcc.gnu.org/onlinedocs/cpp/Macro-Arguments.html][gcc documentation]] pointing up arguments to function-like C macros
  /must be valid C identifiers/. A lisp macro really *is* an ordinary
  function but only evaluated in macro expansion phase. And Lisp code
  is perfectly valid parameter for such a function.
